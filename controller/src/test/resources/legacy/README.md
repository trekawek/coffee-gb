# Legacy snapshot fixtures

The bounded migration reader supports snapshots emitted by Coffee GB 1.7.13 and 1.7.14. That
window is explicit: older descriptors are rejected unless a reviewed historical shape and fixture
are added deliberately.

- `coffee-gb-1.7.13-cpu-instrs.sn`: tag `coffee-gb-1.7.13`, commit
  `25391cc318d99474976d3762bf5d7cd36777596d`, SHA-256
  `7a7ba6fa23538fcd2bdb734487a3b30a18452b69379e18712fc289858ba191ec`.
- `coffee-gb-1.7.14-cpu-instrs.sn`: tag `coffee-gb-1.7.14`, commit
  `53c27ff2360f7f903b8c8a69f92c7d9463d4a207`, SHA-256
  `3588e825efd91f8555d2fd941645a21af1c2ba330df6b56053f01ed81faf5573`.

The tagged build loaded `controller/src/test/resources/roms/cpu_instrs.gb` with bootstrap mode
`SKIP`, ran 1,000 Game Boy ticks, and serialized `Gameboy.saveToMemento()` through the release's
native `ObjectOutputStream` snapshot path.

These binaries are intentionally committed so migration compatibility is tested against bytes
emitted by prior releases, rather than bytes generated by the current implementation.
